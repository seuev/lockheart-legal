Terms of Service

The rules for using Lockheart — moderation, role security, whispers, and captioning, across every server it runs in.

Last updated 14 June 2026
01

What Lockheart does

Depending on how a server configures it, Lockheart may:

  • Monitor role changes and log them to designated channels.
  • Automatically apply or remove roles based on server-defined rules.
  • Send moderation notices, including bilingual (English/Arabic) messages, to users affected by automated actions.
  • Detect messages sent in designated "restricted" channels and take automated action, including deleting the message, banning the account, and sending the user a direct message explaining the action and how to appeal.
  • Provide a "whisper" feature allowing a user to send a message that only specific, selected recipients can reveal.
  • Generate captioned images and GIFs from a text command (-cap / -caption).
  • Send audit logs of the above actions to channels or webhooks designated by server administrators.
02

Eligibility

You must meet Discord's own minimum age requirements to use Discord, and by extension, Lockheart. Server administrators are responsible for configuring Lockheart in a way that complies with Discord's Terms of Service and Community Guidelines.

03

Server administrator responsibilities

Administrators who configure Lockheart's automated moderation features (including role enforcement and restricted-channel autobans) are solely responsible for:

  • Ensuring these features comply with Discord's Terms of Service.
  • Clearly communicating server rules to members, including any channels where sending a message results in an automatic ban.
  • Providing and maintaining a working appeals process for users affected by automated bans.
  • Reviewing appeals submitted by affected users in good faith.

Automated bans are reversible by server staff. If you believe you were banned in error, the message you receive from Lockheart will direct you to an appeal server where you can open a ticket.

04

Acceptable use

You agree not to use Lockheart to:

  • Harass, threaten, or abuse other users, including through the whisper feature.
  • Generate captioned content that violates Discord's Community Guidelines, including hate speech, harassment, or sexually explicit material involving minors.
  • Attempt to exploit, reverse-engineer, or disrupt the Bot's functionality.
  • Circumvent server moderation actions taken by Lockheart.
05

Whisper feature

The whisper feature allows a user to compose a message and image visible only to selected recipients, who must actively choose to reveal it. Senders are responsible for the content of whispers they send. Server staff may review whisper metadata (sender, recipients, timestamps) through audit logs for moderation purposes, but message content is not stored or logged.

01

Information Lockheart accesses

To operate, Lockheart receives data from Discord's API about servers it is added to, including:

  • Usernames, user IDs, avatars, and banners of members.
  • Role assignments and role changes.
  • Messages sent in channels Lockheart can see, where required for a specific feature (for example, detecting the -cap command or messages sent in a restricted channel).
  • Invite usage data, to determine which invite a new member used to join.
  • Attachments and image/GIF URLs, when used with the captioning feature.
02

What is logged

Server administrators can configure Lockheart to send activity logs to specific channels or external webhooks. These logs may include:

EventData logged
Role / gender-role changesUsername, user ID, role change, timestamp
New member joinsUsername, user ID, avatar, invite used, timestamp
Automated role enforcementUsername, user ID, action taken, timestamp
Restricted-channel autobansUsername, user ID, channel, message content (if any), reason, timestamp
Whisper sentSender, recipients, whether an image was attached, timestamp — not the message text
Whisper revealedWhich recipient revealed it, and when

Whisper message content is never logged or stored. Once all recipients have revealed a whisper, Lockheart deletes its in-memory record of that whisper entirely. Audit logs only ever contain metadata — who sent it, who received it, whether it had an image, and when it was revealed — never the message text itself.

03

How long data is kept

  • Whisper data is held only in memory while a whisper is pending, and is deleted once all recipients have revealed it. If the Bot restarts before this happens, pending whisper data is lost.
  • Audit logs are sent to channels or webhooks chosen by the server's administrators. Retention of these logs is controlled by the server, not by Lockheart.
  • Captioned images/GIFs are generated on demand and are not stored after being sent.
  • Lockheart does not maintain its own persistent database of user profiles.
04

Restricted channel / autoban feature

If a server configures a restricted "do not post here" channel, any message sent there by a member without an exempt role will result in: deletion of the message, an automated ban, a direct message to the user explaining the action and how to appeal, and a log entry as described above (including the message content, if the message was text).

This feature is configured per-server by that server's administrators. Lockheart does not decide which channels are restricted — server staff do.

05

Direct messages

Lockheart may send a user a direct message in limited, specific circumstances — for example, to notify them of an automated ban and provide an appeal link. Lockheart does not read or store the content of DMs sent to it by users.

06

Data sharing

Lockheart does not sell or share data with third parties for advertising or marketing purposes. Audit log data is only sent to destinations (channels or webhooks) explicitly configured by each server's administrators.

07

Your rights

Because Lockheart does not maintain a central database of user data outside of Discord itself and server-configured logs, requests to access or delete data should be directed to the administrators of the specific server where you interact with Lockheart. Removing the Bot from a server, or leaving the server, stops Lockheart from accessing your data in that server going forward.
